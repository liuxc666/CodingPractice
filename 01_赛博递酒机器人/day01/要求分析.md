### 使用ai查询的资料：

#### 疑问1：在 .action 文件中，如何一次性将数组传过去？

在 ROS 2 的接口定义语言（IDL）中，定义数组非常简单，只需要在类型后面加上 **[]** 即可。
```
int32[] target
```

#### 疑问2、 Server 端的“任务队列（Task Queue）”如何实现？

在标准的 ROS 2 Action Server 中，默认的模板通常是“单任务”的（一个任务来，执行，结束）。如果你想用 std::vector 来缓存多个 Goal 并挨个执行，你在 C++ 中需要这样构建你的 Server 框架：

#### 1. 定义缓存队列

在你的 Action Server 类中，定义一个用来存放 Goal Handle 的 Vector（或者更推荐用 [[更好的队列容器|std::queue 队列]]，遵循先进先出 FIFO）：

```cpp
// 引入 Action 的 ServerGoalHandle 类型
using GoalHandleMyAction = rclcpp_action::ServerGoalHandle<MyAction>;

// 用来缓存客户端发来的所有任务句柄
std::vector<std::shared_ptr<GoalHandleMyAction>> goal_queue_;
```

#### 2. 在 handle_accepted 回调中“只排队，不执行”

当 Client 发来一个新目标，Server 批准后会触发 handle_accepted。在这里，你**不要**直接去执行它，而是把它“啪”地一下扔进 Vector 里：

```cpp
void handle_accepted(const std::shared_ptr<GoalHandleMyAction> goal_handle)
{
    // 把新任务塞进队列尾部
    goal_queue_.push_back(goal_handle);
    RCLCPP_INFO(this->get_logger(), "新任务已排队，当前队列长度: %zu", goal_queue_.size());
}
```

#### 3. 独立的“工作线程（Worker Thread）”挨个消化任务

你需要有一个独立的线程（比如你在第一天提到的 std::thread），它在一个死循环里不断检查 goal_queue_：

- **如果队列为空**：机器人处于 IDLE 状态，闭眼睡觉（sleep）。
    
- **如果队列不为空**：
    
    1. 从 Vector 头取出第一个 goal_handle。
        
    2. 将状态切换为 DELIVERING，开始执行移动和递送逻辑。
        
    3. 执行完毕后，给该任务返回 Result，然后将其从 Vector 中 erase（清除）掉。
        
    4. 继续循环，读取下一个。
